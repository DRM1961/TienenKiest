Instructions for the RPI used as MQTT broker and Display PART2

>>1: make app.py start at boot time
>>first make a service file:
sudo nano /etc/systemd/system/mqtt_dashboard.service
>> paste below content but change /home/rpi6/dev/TienenKiest to your development directory path
>> and change the user from rpi6 to your username
[Unit]
Description=MQTT Dashboard Service
After=network.target

[Service]
ExecStart=/usr/bin/python3 /home/rpi6/dev/TienenKiest/app.py
WorkingDirectory=/home/rpi6/dev/TienenKiest
Restart=always
User=rpi6 

[Install]
WantedBy=multi-user.target

>> (to save, press CTRL-X, Y and then Enter)

>>enable and start the service


>>2: open browser window on startup
>>check if autostart file exists:
ls /home/rpi6/.config/lxsession/LXDE-pi/autostart

>>if the file is not listed then we copy it from elsewhere:
mkdir /home/rpi6/.config/lxsession
mkdir /home/rpi6/.config/lxsession/LXDE-pi
cp /etc/xdg/lxsession/LXDE-pi/autostart /home/rpi6/.config/lxsession/LXDE-pi/

>>now edit the autostart file:
sudo nano /home/rpi6/.config/lxsession/LXDE-pi/autostart

>>move cursor to end and add:
@chromium-browser --kiosk http://localhost:5000


>> (to save, press CTRL-X, Y and then Enter)

>> reboot to verify changes are ok
